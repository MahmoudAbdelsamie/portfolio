Session-Based Authentication System

The Session-Based Authentication System is a secure user authentication system that uses session cookies to manage user logins and maintain session integrity. It is designed to ensure secure access control and user data protection while providing a seamless user experience.

Key Features:

  • User Registration and Login: Allows users to register and log in securely, storing session data in cookies.
  • Session Management: Manages user sessions, ensuring that only authenticated users can access protected resources.
  • Secure Password Handling: Implements bcrypt for secure password hashing and storage.
  • CSRF Protection: Utilizes tokens to protect against Cross-Site Request Forgery attacks.
  • Logout Functionality: Provides secure logout functionality to invalidate user sessions.
  • Access Control: Implements role-based access control to restrict access to specific resources based on user roles.

Technologies Used:

  • Backend: Node.js, Express.js
  • Database: PostgreSQL
  • Security: bcrypt, CSRF tokens
  • Session Management: Express-session, cookie-parser

JWT-Based Authentication System

Key Features:

  • User Registration and Login: Allows users to register and log in securely, generating JWTs for session management.

  • Secure Password Handling: Utilizes bcrypt for secure password hashing and storage.
  • Logout Functionality: Provides secure logout functionality to invalidate tokens and terminate sessions.
